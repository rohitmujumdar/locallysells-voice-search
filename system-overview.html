The three platforms, and what each one does

Voice spine
Vapi
Runs the live call. Hears the caller (speech to text), speaks back (text to speech), and orchestrates the loop. Calls out to the other two for thinking and data.
The brain
Nebius
Hosts the LLM (Llama 3.3 70B) that decides what to do, e.g. "I should search the catalog." Paid by Nebius credits, not Vapi credits. Swapped in as Vapi's custom LLM.
Tool host
Insforge
Runs our search + order code as a serverless edge function. Stable public URL, off your laptop. This is the only piece we actually wrote.

The 30-second mental model

Vapi runs the live phone call. The thinking is done by an LLM on Nebius. When that LLM needs real product info, it calls our tools on Insforge, which query a copy of the LocallySells catalog and return matches with prices, or place an order. Everything else about LocallySells (the database, payments, delivery) stays a black box the agent never touches.

The two tools

What the agent can call, and what comes back

search_products → answer

// LLM args
{ "query": "marlboro red",
  "category": "cigarettes" }

// returned (spoken)
"Found 2 matches. Top results:
 - Marlboro Red King Size — $8.99 — Pack
 - Marlboro Red Carton — $84.99"

place_order → confirmation

// LLM args (stateless: full list)
{ "items": [
    {"name":"marlboro red","quantity":2},
    {"name":"zyn cool mint","quantity":1} ],
  "delivery_address": "123 Main St, LA" }

// returned (spoken)
"Order LS-HERM01 confirmed. Total
 $23.47. ETA ~45 min. 21+ ID at door."

Both tools live at the same Insforge URL; the function reads the tool name from the Vapi request and routes. place_order is stateless: the agent gathers the items through the conversation and passes the whole list at once, so there's no cart state for a serverless platform to lose.

Search is semantic: every product is embedded once with Nebius (Qwen3-Embedding-8B, 512-dim, precomputed). At query time the function embeds the caller's phrase and ranks the catalog by cosine similarity, so "a smooth light cigarette" finds Marlboro Lights and "an easy starter vape" finds the SMOK Novo kit. A keyword-hybrid boost keeps exact brand mentions on top, and it falls back to keyword search if embeddings ever fail.
